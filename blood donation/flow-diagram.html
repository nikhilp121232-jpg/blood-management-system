🩸 Life Saver - Data Flow Architecture

Complete Integration between Forms and Admin Panel

👥 Donor Registration Flow

1
📝 User Opens Form
User navigates to "Register as Donor" page
2
✍️ Fill Details
Name, Blood Type, Age, Phone, City, Email
3
✅ Validation
Form validates required fields and blood type
4
💾 Save to Storage
Data saved to localStorage with unique ID
5
🎉 Success Message
User sees confirmation with Donor ID

🏥 Blood Request Flow

1
📝 User Opens Form
User navigates to "Request Blood" page
2
✍️ Fill Details
Patient, Hospital, Blood Type, Units, Urgency
3
✅ Validation
Form validates required fields and urgency
4
💾 Save to Storage
Request saved to localStorage with unique ID
5
🎉 Success Message
User sees confirmation with Request ID

👨‍💼 Admin Panel Flow

1
🚀 Panel Opens
Admin opens the management panel
2
📥 Load Data
Reads donors and requests from localStorage
3
📊 Display Data
Shows all data in tables and cards
4
🔄 Auto-Refresh
Updates every 2 seconds automatically
5
⚡ Real-time Updates
New data appears without manual refresh

💾 Central Data Storage (localStorage)

All data is stored in browser's localStorage for persistence

lifesaver_donors Stores all registered donors
lifesaver_requests Stores all blood requests

✨ Key Features

🔄
Real-time Sync
Admin panel automatically refreshes every 2 seconds
🆔
Unique IDs
Auto-generated IDs for donors (D001) and requests (REQ-001)
💾
Persistent Storage
Data remains even after browser close
✅
Validation
Complete form validation before submission
🔍
Search & Filter
Admin can search and filter donors by blood type
🤖
AI Matching
Automatic donor matching based on blood compatibility
